# 13-05-26_06 — PDF не изменился: жёсткая диагностика и исправление ссылки

## Ситуация

Проблема осталась: при нажатии:

```text
Оптимизатор маршрутов → Открыть презентацию PDF
```

открывается не настоящий файл презентации, а какой-то старый/заглушечный PDF с текстом:

```text
Route Optimizer
PDF presentation RU
Automation for routes and reports
Contact: Roman11092017@gmail.com
```

При этом прямые ссылки открываются нормально:

```text
http://localhost:8000/assets/presentations/route_optimizer_en.pdf
http://localhost:8000/assets/presentations/route_optimizer_ru.pdf
```

## Главный вывод

Если прямой адрес файла открывает правильный PDF, значит проблема почти точно **не в самом PDF-файле**.

Проблема в одном из трёх мест:

1. русская кнопка `Открыть презентацию PDF` ссылается на другой файл;
2. на кнопку повешен старый JavaScript-обработчик, который открывает заглушку;
3. браузер/локальный сервер показывает старую версию `index.html` или `script.js` из кэша.

Поэтому сейчас нужно не «ещё раз поменять ссылку на глаз», а найти **фактический источник**, из которого открывается неправильный PDF.

---

# 1. Сначала найти источник старой заглушки

Открой PowerShell и выполни:

```powershell
cd "D:\MyProgect\portfolio-site"

Get-ChildItem -Recurse -File | Select-String -Pattern "Route Optimizer|PDF presentation RU|Automation for routes and reports|route_optimizer|presentation|pdf" | Select-Object Path, LineNumber, Line
```

## Что это делает

Команда ищет по всему проекту строки:

- `Route Optimizer`
- `PDF presentation RU`
- `Automation for routes and reports`
- `route_optimizer`
- `presentation`
- `pdf`

То есть она должна показать, **где именно в проекте спрятана старая заглушка** или старая ссылка.

## Что нужно сделать с результатом

Если команда покажет строки, где есть:

```text
Route Optimizer
PDF presentation RU
Automation for routes and reports
```

то именно этот файл и это место, скорее всего, открывает неправильную презентацию.

Особенно внимательно смотри файлы:

```text
index.html
script.js
main.js
app.js
```

# 2. Быстрая проверка через браузер

Открой сайт:

```text
http://localhost:8000
```

Нажми правой кнопкой мыши на кнопку:

```text
Открыть презентацию PDF
```

Дальше:

```text
Inspect / Просмотреть код
```

Посмотри, что у кнопки внутри.

## Вариант А — это ссылка `<a>`

Если увидишь что-то вроде:

```html
<a href="...">Открыть презентацию PDF</a>
```

то у русской версии должно быть строго так:

```html
<a class="btn btn-primary" href="./assets/presentations/route_optimizer_ru.pdf?v=2026-05-13-06" target="_blank" rel="noopener">
  Открыть презентацию PDF
</a>
```

У английской версии:

```html
<a class="btn btn-primary" href="./assets/presentations/route_optimizer_en.pdf?v=2026-05-13-06" target="_blank" rel="noopener">
  Open PDF presentation
</a>
```

## Вариант Б — это кнопка `<button>`

Если увидишь что-то вроде:

```html
<button onclick="openRoutePresentation()">Открыть презентацию PDF</button>
```

или:

```html
<button data-pdf="...">Открыть презентацию PDF</button>
```

значит ссылку открывает JavaScript, и нужно исправлять не только HTML, но и функцию в `script.js`.

# 3. Самое надёжное исправление через одну функцию

Лучше сделать так, чтобы все PDF открывались только через один центральный список ссылок.

В файле:

```text
D:\MyProgect\portfolio-site\script.js
```

нужно найти место, где открываются PDF, и привести к такой логике.

## Добавить или заменить блок ссылок

```javascript
const PRESENTATION_FILES = {
  ru: {
    route: './assets/presentations/route_optimizer_ru.pdf?v=2026-05-13-06',
    digest: './assets/presentations/client_digest_hub_ru.pdf?v=2026-05-13-06'
  },
  en: {
    route: './assets/presentations/route_optimizer_en.pdf?v=2026-05-13-06',
    digest: './assets/presentations/client_digest_hub_en.pdf?v=2026-05-13-06'
  }
};
```

## Добавить или заменить функцию открытия PDF

```javascript
function openPresentation(projectKey) {
  const lang = currentLang || 'ru';
  const url = PRESENTATION_FILES[lang]?.[projectKey];

  if (!url) {
    alert('PDF presentation file is not configured.');
    return;
  }

  window.open(url, '_blank', 'noopener');
}
```

## Кнопка для русского оптимизатора маршрутов должна вызывать

```html
<button class="btn btn-primary" type="button" onclick="openPresentation('route')">
  Открыть презентацию PDF
</button>
```

## Кнопка для английского оптимизатора маршрутов тоже вызывает ту же функцию

```html
<button class="btn btn-primary" type="button" onclick="openPresentation('route')">
  Open PDF presentation
</button>
```

Разница будет не в кнопке, а в текущем языке сайта `currentLang`.

# 4. Временное самое простое решение без JavaScript

Если Codex опять путается, можно сделать проще: заменить кнопку на обычную ссылку.

Для русской версии:

```html
<a class="btn btn-primary" href="./assets/presentations/route_optimizer_ru.pdf?v=2026-05-13-06" target="_blank" rel="noopener">
  Открыть презентацию PDF
</a>
```

Для английской версии:

```html
<a class="btn btn-primary" href="./assets/presentations/route_optimizer_en.pdf?v=2026-05-13-06" target="_blank" rel="noopener">
  Open PDF presentation
</a>
```

Это самый простой и надёжный вариант.

Если сайт статический, для открытия PDF **не нужна отдельная JavaScript-функция**. Обычная ссылка работает стабильнее.

# 5. Удалить старую PDF-заглушку или старый генератор PDF

Поиск из пункта 1 может показать, что в проекте есть код, который сам создаёт заглушечный PDF.

Ищи такие подозрительные элементы:

```javascript
new Blob
application/pdf
Route Optimizer
PDF presentation RU
generatePdf
openPdf
openRoutePdf
createPresentation
```

Если где-то есть логика, которая создаёт PDF из текста:

```javascript
Route Optimizer
PDF presentation RU
Automation for routes and reports
```

её нужно удалить или больше не использовать.

Для сайта-портфолио правильная логика проще:

```text
кнопка → открыть готовый PDF-файл из папки assets/presentations
```

А не так:

```text
кнопка → сгенерировать какой-то PDF-заглушку в JavaScript
```

# 7. Проверка после исправления

Открой консоль браузера:

```text
F12 → Console
```

Вставь команду:

```javascript
[...document.querySelectorAll('a[href*=".pdf"]')].map(a => a.href)
```

Должно быть видно примерно такие ссылки:

```text
http://localhost:8000/assets/presentations/route_optimizer_ru.pdf?v=2026-05-13-06
http://localhost:8000/assets/presentations/route_optimizer_en.pdf?v=2026-05-13-06
```

Если кнопка сделана через `onclick`, выполни ещё:

```javascript
[...document.querySelectorAll('[onclick]')].map(el => ({
  text: el.textContent.trim(),
  onclick: el.getAttribute('onclick')
}))
```

Если там есть старые функции типа:

```text
openRoutePdf
openRoutePresentation
createPdf
```

их нужно проверить в `script.js`.
