# 13-05-26_05 — Исправление ссылки на PDF-презентацию Route Optimizer RU

## Что случилось

Сейчас при нажатии:

```text
Оптимизатор маршрутов → Открыть презентацию PDF
```

в русской версии открывается не тот PDF-файл. Вместо реальной презентации открывается какой-то служебный/заглушечный PDF с текстом:

```text
Route Optimizer
PDF presentation RU
Automation for routes and reports
Contact: Roman11092017@gmail.com
```

При этом сами нужные файлы по прямым ссылкам доступны:

```text
http://localhost:8000/assets/presentations/route_optimizer_ru.pdf
http://localhost:8000/assets/presentations/route_optimizer_en.pdf
```

Значит, проблема почти точно не в самих PDF-файлах, а в том, что кнопка в русской версии сайта ссылается не на тот путь, либо JavaScript подставляет старую ссылку/заглушку.

## Шаг 4. Как должна выглядеть правильная логика в `script.js`

Если у тебя сайт сделан через объект с данными проектов, логика должна быть примерно такой:

```javascript
const projects = {
  route: {
    pdf: {
      ru: "assets/presentations/route_optimizer_ru.pdf",
      en: "assets/presentations/route_optimizer_en.pdf"
    }
  },
  digest: {
    pdf: {
      ru: "assets/presentations/client_digest_hub_ru.pdf",
      en: "assets/presentations/client_digest_hub_en.pdf"
    }
  }
};
```

А сама кнопка должна получать ссылку так:

```javascript
const currentLang = document.documentElement.lang === "en" ? "en" : "ru";
const pdfUrl = projects.route.pdf[currentLang];

pdfButton.href = pdfUrl;
pdfButton.target = "_blank";
pdfButton.rel = "noopener noreferrer";
```

Если используется `window.open`, тогда так:

```javascript
pdfButton.addEventListener("click", function () {
  const currentLang = document.documentElement.lang === "en" ? "en" : "ru";
  window.open(projects.route.pdf[currentLang], "_blank", "noopener,noreferrer");
});
```

Но лучше использовать обычный `<a href="...">`, потому что это проще и надежнее.

## Шаг 5. Как должна выглядеть кнопка в HTML

Для статической кнопки можно сделать так:

```html
<a
  class="action-button"
  id="routePdfButton"
  href="assets/presentations/route_optimizer_ru.pdf"
  target="_blank"
  rel="noopener noreferrer"
>
  Открыть презентацию PDF
</a>
```

А JavaScript при переключении языка будет менять `href`:

```javascript
routePdfButton.href = currentLang === "ru"
  ? "assets/presentations/route_optimizer_ru.pdf"
  : "assets/presentations/route_optimizer_en.pdf";
```

## Справочник терминов и сокращений

**PDF** — файл-презентация или документ, который открывается в браузере или отдельной программе.

**href** — адрес ссылки в HTML. Именно он определяет, куда ведет кнопка-ссылка.

**target="_blank"** — команда открыть ссылку в новой вкладке.

**rel="noopener noreferrer"** — безопасный режим открытия внешней/новой вкладки.

**script.js** — файл JavaScript, где обычно хранится логика переключения вкладок, языка и ссылок.

**index.html** — основной HTML-файл сайта.

**кэш браузера** — временное хранилище браузера. Иногда браузер показывает старую версию файла, даже если ты уже заменил файл на компьютере.

**Ctrl + F5** — жесткое обновление страницы без использования старого кэша.

**Blob PDF / data URL** — способ динамически создавать файл прямо в браузере. В данном случае это не нужно, потому что PDF должен открываться из папки `assets/presentations/`.
