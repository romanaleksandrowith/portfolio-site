# 2. Важное решение по контактам

Раньше была просьба сделать контакты в виде кнопок. Сейчас проще и правильнее сделать **один аккуратный информационный блок**.

## Как должно быть

В разделе `Обо мне и контакты` должен быть один блок:

```text
Контакты

E-mail: Roman11092017@gmail.com
Telegram: @W10FAN
GitHub: github.com/romanaleksandrowith
Portfolio: romanaleksandrowith.github.io/portfolio-site
```

## Как не должно быть

Не надо делать так:

```text
[ E-mail ] [ Telegram ] [ GitHub ] [ Portfolio ]
```

Почему: для потенциального клиента это выглядит как набор лишних кнопок. Лучше сделать спокойно, понятно и по-деловому.

# 6. Важное замечание по CSS-блоку

CSS-блок выше может не сработать на 100%, если в твоём проекте классы называются по-другому.

Например если блок `Практичная автоматизация` имеет класс:

```html
<div class="practical-box">
```

тогда нужно добавить этот класс в общий список:

```css
.practical-box {
  background: ...
}
```

Или проще — в HTML добавить ему общий класс:

```html
<div class="practical-box info-card">
```

Правильная логика такая:

```text
Все большие карточки с текстом получают класс info-card.
Все маленькие карточки внутри них получают класс inner-card.
```

Тогда дизайн будет единым.

## Шаг 5. Проверь градиенты

Градиент должен быть у всех основных блоков:

- [ ] `о работе`
- [ ] `документы по шаблону`
- [ ] `Практичная автоматизация`
- [ ] `Что уже есть`
- [ ] `Куда можно развить`
- [ ] `Для кого полезно`
- [ ] `Обо мне`
- [ ] `Контакты`

## Шаг 6. Проверь внутренние подблоки

Внутренние подблоки должны быть значительно темнее внешних карточек.

Например блок:

```text
01
Оптимизатор маршрутов
Расчет порядка точек, расстояния, времени, карты и отчетов...
```

должен быть тёмным, почти как тёмная карточка внутри синего блока.

## Шаг 7. Проверь телефон

Открой инструменты разработчика в браузере:

```text
F12 → Toggle device toolbar
```

Проверь ширины:

```text
360px
390px
430px
```

На телефоне:

- [ ] текст не выходит за края;
- [ ] заголовки не огромные;
- [ ] карточки идут в одну колонку;
- [ ] контакты читаются нормально;
- [ ] меню не перекрывает контент.

# 8. Что делать, если после Codex часть блоков опять не градиентная

Это почти точно значит, что у этих блоков другие CSS-классы.

Тогда нужно написать Codex вот так:

```text
Найди все блоки на странице, которые содержат информационный текст, но не имеют градиента.

Особенно:
- о работе
- документы по шаблону
- Практичная автоматизация

Для каждого такого блока:
1. добавь класс info-card;
2. если внутри есть маленькие карточки, добавь им класс inner-card;
3. не меняй текст;
4. не меняй структуру меню;
5. не удаляй RU/EN.
```

# 13. Справочник терминов и сокращений

| Термин | Объяснение |
|---|---|
| `CSS` | Файл стилей сайта: цвета, размеры, отступы, карточки, кнопки. |
| `HTML` | Структура страницы: блоки, заголовки, тексты, ссылки. |
| `JS` / `JavaScript` | Логика сайта: переключение вкладок, языков, кнопок. |
| `class` | Имя элемента в HTML, по которому CSS понимает, как его оформить. |
| `info-card` | Предлагаемый общий класс для больших информационных карточек. |
| `inner-card` | Предлагаемый общий класс для маленьких тёмных карточек внутри больших блоков. |
| `gradient` / градиент | Плавный переход цвета, например от светло-синего к тёмно-синему. |
| `clamp()` | CSS-функция, которая задаёт адаптивный размер: минимум, гибкий размер и максимум. |
| `media query` | CSS-правило для отдельной настройки сайта под телефон или планшет. |
| `GitHub Pages` | Публикация статического сайта из GitHub-репозитория. |
| `manual/` | Локальная папка для твоих `.md`-инструкций, которую не надо отправлять на GitHub. |
